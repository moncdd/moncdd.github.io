8.3. REST

REST es un servicio web que se comunica a través de la WWW. Está basado en REST Architecture, ofrece comunicación entre aplicaciones de software que se ejecutan en diferentes plataformas y entornos. Un servicio web REST es una función o método que se puede llamar enviando una solicitud HTTP a una URL, y el servicio devuelve el resultado como respuesta.

REST (REpresentational State Transfer) significa Transferencia Estatal REpresentational. El objetivo principal de los servicios web REST es hacer que los servicios web sean más eficaces. Los servicios web REST tratan de definir los servicios utilizando los diferentes conceptos que ya están presentes en HTTP. REST es un enfoque arquitectónico, no un protocolo.

No define el formato estándar de intercambio de mensajes. Podemos construir servicios REST con XML y JSON. JSON es más popular con REST. La abstracción clave es un recurso en REST. Un recurso puede ser cualquier cosa. Se puede acceder a través de un identificador de recursos uniformes (URI). Los métodos importantes de HTTP son:

  • GET: Lee un recurso.
  • PUT: Actualiza un recurso existente.
  • POST: Crea un nuevo recurso.
  • DELETE: Eliminó el recurso.

HTTP también define el siguiente código de estado estándar:

404: RESOURCE NO FOUND
200: SUCCESS
201: CREATED
401: UNAUTHORIZED
500: SERVER ERROR

Las restricciones del servicio REST son:

  • Debe haber un productor de servicios y un consumidor de servicios.
  • El servicio es sin estado.
  • El resultado del servicio debe ser cacheable.
  • La interfaz es uniforme y expone los recursos.
  • El servicio debe asumir una arquitectura en capas.

Las ventajas de los servicios web RESTful son:

  • Los servicios web REST son independientes de la plataforma.
  • Se puede escribir en cualquier lenguaje de programación y se puede ejecutar en cualquier plataforma.
  • Proporciona diferentes formatos de datos como JSON, texto, HTML y XML.
  • Es rápido en comparación con SOAP porque no hay una especificación estricta como SOAP.
  • Son reutilizables.
  • Son realizados en lenguaje neutral.
