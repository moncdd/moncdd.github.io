8.4. SOAP vs REST

SOAP Protocol REST Services
SOAP es un protocolo. REST es un enfoque arquitectónico.
Acrónimo SOAP para el Protocolo de Acceso de Objetos Simples. Acrónimo REST para la Transferencia Estatal REpresentacional.
En SOAP, el formato de intercambio de datos es siempre XML. No existe un formato estricto de intercambio de datos. Podemos usar JSON, XML, etc.
XML es el formato de intercambio de datos más popular en los servicios web SOAP. JSON es el formato de intercambio de datos más popular en los servicios web de REST.
SOAP utiliza el lenguaje de definición de servicio web (WSDL). REST no tiene ningún idioma de definición estándar.
SOAP no plantea ninguna restricción al transporte. Podemos usar HTTP o MQ. Los servicios REST utilizan el protocolo HTTP más popular.
Los servicios web SOAP son típicos de implementar. Los servicios REST son más fáciles de implementar que SOAP.
Los servicios web SOAP utilizan la API JAX-WS. Los servicios web REST utilizan la API JAX-RS.
El protocolo SOAP define demasiados estándares. Los servicios REST no hacen hincapié en demasiados estándares.
SOAP no puede usar servicios REST porque es un protocolo. El servicio REST puede utilizar los servicios web de SOAP porque es un enfoque arquitectónico que puede utilizar cualquier protocolo como HTTP y SOAP.
Las lecturas de SOAP no se pueden cachear. Las lecturas REST se pueden cachear.
