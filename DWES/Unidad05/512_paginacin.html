5.12. Paginación

PHP se utiliza principalmente para almacenar y mostrar datos de una base de datos. Es posible que una consulta SELECT devuelva millones de tuplas. No es una buena idea mostrar todas las tuplas en una sola página. Una gran lista de registros en una sola página puede tomar tanto tiempo para cargar la página y también consumir tiempo para encontrar datos específicos. Esto puede causar confusión en el usuario. Por lo tanto, se recomienda dividir los registros en varias páginas de acuerdo con las peticiones del usuario.

La paginación es una forma de mostrar los datos en varias páginas en lugar de ponerlos en una sola página. La paginación ayuda a dividir los registros en varias páginas, lo que hace que los datos sean más legibles y comprensibles. La paginación es una tarea común para los desarrolladores de PHP. MySQL ayuda al desarrollador a crear la paginación mediante el uso de la cláusula LIMIT, que toma dos argumentos. El primer argumento como OFFSET y el segundo argumento es el número de registros que volverán de la base de datos.

5.12.1. Ventajas y desventajas de la paginación

  • Ventajas de la Paginación
    • La paginación es muy útil en proyectos a gran escala porque hace que el trabajo web sea más profesional. No sólo más profesional, sino que también hace que la página web funcione mucho más rápido, preciso y eficiente.
    • Con la ayuda de la paginación, podemos guardar el tiempo de carga de una página dividiendo los datos en varias páginas. Nos ahorra cargar mucha información a la vez.
    • El uso de la paginación mejora la experiencia del usuario, los ingresos publicitarios y la disminución del tiempo de carga de la página.
  • Desventajas de la Paginación
    • La propia paginación es un gran sobrecarga en PHP, que es una de las desventajas de la paginación.
    • La paginación puede ser la causa del bajo ranking en los motores de búsqueda porque cuando una página está lejos de la página de inicio y requiere varios clics, por lo general no consigue un rango alto de página.
    • También limita el número de enlaces, acciones sociales, número total de resultados a visibles en la página web, y el texto de anclaje que una página recibe cuando la información se divide en varias páginas.

5.12.2.

Para implementar la paginación, requiere un gran conjunto de datos para aplicar paginación a ella. Por lo tanto, primero necesitamos crear una base de datos y una tabla. Después de eso, proporcionar los registros en la tabla y comenzar a codificar para crear paginación. Para que los datos que se obtienen de la base de datos se pueden dividir a través de varias páginas.

Los pasos para crear paginación son:

  1. Crear una base de datos y una tabla.
  2. Proporcionar una lista de los registros en la tabla.
  3. Conectar con la base de datos MySQL.
  4. Crear el enlace de paginación para dividir los datos en varias páginas y añadirlos a la parte inferior de la tabla.
  5. Traer datos de la base de datos y visualizarlos en las múltiples páginas.

Lo más importante para realizar la paginación es la sentencia SQL de MySQL:

$query = "SELECT *FROM table LIMIT " . $page_first_result . ',' . $results_per_page;
