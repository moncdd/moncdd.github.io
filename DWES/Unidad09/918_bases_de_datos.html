9.1.8. Bases de datos

Aunque se pueden usar otros, Doctrine es el ORM por defecto de Symfony. La configuración de la base de datos se introduce en el fichero .env

Los ORM liberan al programador de muchas tareas, generando el código necesario para comunicarse con la base de datos. Proporcionan un nivel de abstracción adicional que permite integrar diferentes orígenes de datos con un lenguaje común y facilita la reusabilidad del código.

El elemento básico de Doctrine son las entidades. Las entidades son clases que están asociadas con la base de datos. Los atributos de estas clases son persistentes, es decir, se almacenan en una base de datos.

La relación entre entidades y la base de datos se describe mediante metadatos que pueden añadirse con anotaciones en la propia entidad o en un fichero externo. En general, las anotaciones tienen que especificar:

  • La tabla con la que está mapeada la entidad
  • La columna con la que está mapeado cada atributo persistente
  • Las modificaciones de cada atributo (p.e. AUTO_INCREMENT)
  • Las asociaciones con otras entidades. Las asociaciones son la contraparte de las foreign key.

Hay varios tipos de asociaciones que se diferencia por:

  • Cardinalidad: muchos a uno, uno a muchos, uno a uno, muchos a muchos
  • Direccionalidad: unidireccionales o bidireccionales
  • Si incluyen una referencia a la propia entidad

Las consultas que se pueden realizar se obtienen de un repositorio. Algunas de las consultas disponibles son:

  • findBy: utiliza criterios de búsqueda que se introducen en un array
  • findOneBy: igual que el anterior pero devolviendo un solo resultado
  • findAll: devuelve todas las filas de una tabla asociada

Doctrine también tiene su propio lenguaje de consultas, se llama Doctrine Query Language (DQL) y es bastante parecido a SQL.
