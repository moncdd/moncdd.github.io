9.1.2. Estructura de directorios

Al crear un nuevo proyecto con el comando:

symfony new my_project_directory --webapp

se crea una jerarquía de directorios. Si no indicamos el atributo --webapp crea otra estructura más sencilla pues no indicamos que deseamos crear una aplicación web.

En Symfony, la ubicación de los ficheros es muy importante. Se espera que cada tipo de componente esté en un directorio concreto. La estructura de directorios de un proyecto recién creado es:

Los directorios más importantes son:

  • El directorio raíz del proyecto. Aquí está el fichero .env donde se almacena la configuración de base de datos y correo.
  • /config/packages. Está el fichero security.yaml que contiene la configuración de seguridad
  • /src/Controller. Están los controladores de la aplicación
  • /src/Entity. Se utiliza para guardar las entidades
  • /src/Templates. Se utiliza para las plantillas
  • /assets. Se ubican los ficheros que se necesitan mandar al cliente, como js, css...
  • /public. Aquí se encuentra la página pública que se accede vía URL en el navegador
