7.2. Generación dinámica de páginas interactivas

7.2.1. Separación de la lógica de negocio

Uno de los temas principales en el diseño de aplicaciones es la reusabilidad del código. Para conseguirlo, las aplicaciones se dividen en partes que se relacionan entre si, pero que también tienen sentido de manera independiente.

A pequeña escala, se crean funciones y clases que se pueden reu­tilizar en otras aplicaciones. A nivel de diseño de aplicaciones, se plantea una arquitectura con varios componentes desacoplados que se ocupan de las diferentes partes de la aplicación.

El enfoque que se va a seguir consiste en llevar la lógica de presentación al cliente. Al fin y al cabo, en el cliente es donde se muestra la información. El lenguaje más extendido en el lado del cliente es JavaScript. Ajax es una técnica de diseño de aplicaciones web que se basa en hacer peticiones al servidor desde JavaScript. De esta manera, al desacoplar la lógica de negocio de la presentación se obtienen varias ventajas:

  • El código es más fácil de mantener y modificar, al no estar mezclado
  • El código del servidor se puede reutilizar con otros clientes
  • Se puede trabajar en la parte del cliente y en la del servidor en paralelo

El código JavaScript llega al cliente desde el servidor incluido en HTML, se ejecuta en el navegador y puede manipular la página web para cambiar su aspecto, mostrar mensajes o establecer comunicaciones con el servidor.

Para modificar la página web, JavaScript utiliza el DOM, un estándar del W3C que permite acceder y modificar el contenido, la estructura y el estilo de un documento HTML. Las funciones y atributos básicos que permiten manipular el DOM son:

Función Descripción
document.createElement() Crea un nuevo elemento
document.getElementById() Devuelve el elemento de la página que tiene ese id
element.appendChild() Inserta el elemento como hijo de element
element.innerHTML Propiedad que representa el contenido de un elemento

7.2.2. Obtención remota de información

Hasta ahora, hemos obtenido la información del servidor:

  • Al acceder a  una página introduciendo la URL
  • Al hacer clic en un vínculo
  • Al enviar un formulario
  • Al redirigir una petición

En todos estos casos, el navegador modifica el contenido mostrado al cliente con la respuesta que obtiene del servidor. No obstante, es posible modificar el contenido usando JavaScript, en concreto AJAX. AJAX es el acrónimo de Asynchronous JavaScript And XML (JavaScript asíncrono y XML), una técnica de desarrollo web para crear aplicaciones web asíncronas.

Estas aplicaciones se ejecutan en el cliente, es decir, en el navegador de los usuarios mientras se mantiene la comunicación asíncrona con el servidor en segundo plano. De esta forma es posible interactuar con el servidor sin necesidad de recargar la página web, mejorando la interactividad, velocidad y usabilidad en las aplicaciones.

Los datos adicionales se solicitan al servidor y se cargan en segundo plano sin interferir con la visualización ni el comportamiento de la página, aunque existe la posibilidad de configurar las peticiones como síncronas de tal forma que la interactividad de la página se detiene hasta la espera de la respuesta por parte del servidor.

El elemento central de AJAX es el objeto XMLHttpRequest, que representa una petición al servidor. Utilizándolo es posible hacer una petición al servidor sin que su respuesta se muestre automáticamente como una nueva página en el navegador. En lugar de eso, la respuesta se procesa mediante JavaScript.

7.2.3. Modificación de la estructura y contenido de la página web.

En muchos casos, la respuesta del servidor consiste en una serie de datos que hay que mostrar en la página. Puede ser desde una sección hasta una página completa, introduciendo o eliminando elementos.

Como no siempre se recibe una web completa, se suelen recibir datos en diferentes formatos, desde una simple cadena de texto, hasta datos complejos en formato JSON o XML.

Cuando se popularizó AJAX, era habitual devolver los datos en formato XML (de ahí el nombre de esta tecnología). No obstante, en la actualidad, el formato más extendido en el desarrollo web es JSON porque es más accesible y rápido que XML.
